## Gotchas & Notes for Future Agents

1. **SSH tunnel required for live matrix**: To run the matrix against production, open a tunnel first: `ssh -f -N -L 54320:172.18.0.4:5432 root@89.167.42.128` then set `REMOTE_DATABASE_URL=postgresql://cityecho:CityEcho2026xK9pQ7mN@127.0.0.1:54320/cityecho`.
2. **Docker postgres IP may change**: The IP `172.18.0.4` is the current Docker bridge IP for the `postgres` container. If Coolify recreates the network, this IP will change. Re-check with `docker inspect postgres --format '{{range .NetworkSettings.Networks}}{{.IPAddress}}{{end}}'`.
3. **Coolify auto-deploy is OFF**: All webhook secrets are null. Every deploy must be triggered manually via the Coolify API.
4. **Yandex OAuth and YooKassa are explicitly deferred** — not blocking production, logged as skipped.
5. **Rate limiting can block matrix runner**: If you probe the login endpoint with brute-force tests before running the matrix, the rate limiter (429) will block the UI login in `openPages()` for several minutes.
